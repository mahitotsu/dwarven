# ADR: 主要な設計判断

1. 使用ライブラリ: pandas を採用
   - 理由: CSV読み込みと集計が簡潔に書け、生産性とメンテナンス性が高い。

2. 可視化ツール: matplotlib をデフォルト、plotly をオプションでサポート
   - 理由: matplotlib は静的画像生成に適し、軽量。plotly は必要に応じて追加してインタラクティブ機能を提供する。

3. レポート生成: Jinja2 テンプレートでHTMLを生成
   - 理由: HTMLテンプレートの柔軟性と既存エコシステム（Bootstrap等）を利用可能にするため。

4. CLI実装: argparse を使用
   - 理由: 標準ライブラリで依存を増やさずに柔軟な引数処理が可能。

5. テスト: pytest を採用
   - 理由: Pythonコミュニティ標準であり、fixtureを使ったテストが容易。

将来の拡張:
- 大規模データ向けにdaskやpolarsを検討（パフォーマンス要件が明確になった場合）。
- レポートのホスティングや自動化のためにCI/CDワークフローを追加。
